# TecnoJR Component Library

Welcome to the TecnoJR component documentation! This Storybook showcases our design system, component library, and implementation patterns for the TecnoJR Next.js website.

## üé® Design System

Our design system is built on modern principles:

- **OKLCH Color Space**: Superior perceptual uniformity and wide-gamut support
- **Gradient-First**: Animated gradients using `tecno-purple` ‚Üí `tecno-blue` 
- **Glassmorphism**: Backdrop blur effects with semi-transparent backgrounds
- **Responsive**: Mobile-first design with consistent breakpoints
- **Accessible**: WCAG compliant with keyboard navigation support

### Color Palette

- **tecno-purple** (50-950): Primary brand color - `#9155EB` at 500 level
- **tecno-blue** (50-950): Secondary brand color - `#4A90E2` at 500 level
- **tecno-black** (50-950): Grayscale from white to deep black

## ü§ù Contributing

When adding new components:

1. **Follow naming conventions**: kebab-case files, PascalCase components
2. **Document with JSDoc**: Include description, params, returns, and examples
3. **Create stories**: Show all variants and use cases
4. **Write tests**: Minimum 85% coverage required
5. **Use TypeScript strict**: No `any` types allowed
6. **Maintain consistency**: Check existing patterns in `lib/styles.ts` and `lib/animations.ts`

## üìù Notes

- All components are server components by default - add `'use client'` only when necessary
- Use `@/` path alias for all imports from `src/`
- Prefer Tailwind utilities over inline styles
- Check for existing utility functions before creating new ones
- All gradients use OKLCH color space for better perceptual uniformity
