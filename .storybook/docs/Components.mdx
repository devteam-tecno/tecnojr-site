# Component Catalog

Complete reference of all components in the TecnoJr design system.

## Component Architecture

### Organization Structure
```
src/components/
├── ui/              # Reusable UI primitives
├── layout/          # Layout components (Header, Footer)
├── sections/        # Page sections (Hero, Features, etc.)
└── animated/        # Animation wrappers and effects
```

### Design Principles
1. **Single Responsibility** - Each component does one thing well
2. **Composability** - Components build on primitives
3. **Consistency** - Shared design tokens and patterns
4. **Accessibility** - WCAG 2.1 AA compliance
5. **Performance** - Optimized rendering and animations

## Usage Patterns

### Composing Sections
```tsx
export default function HomePage() {
  return (
    <>
      <HeroSection />
      <FeaturesSection />
      <ServicesSection />
      <AboutSection />
      <ProjectsSection />
      <BudgetSection />
    </>
  );
}
```

### Building Forms
```tsx
<form onSubmit={handleSubmit}>
  <Input
    type="email"
    placeholder="Email"
    {...register("email")}
    aria-invalid={!!errors.email}
  />
  <Textarea
    placeholder="Message"
    {...register("message")}
  />
  <Button type="submit" variant="gradient-primary">
    Send
  </Button>
</form>
```

### Adding Animations
```tsx
import { FadeUp, StaggerFadeUp } from "@/components/animated/motion-wrappers";

<FadeUp>
  <SectionTitle>Heading</SectionTitle>
</FadeUp>

{items.map((item, index) => (
  <StaggerFadeUp key={item.id} index={index}>
    <Card>{item.content}</Card>
  </StaggerFadeUp>
))}
```

## Component Guidelines

### When to Create a New Component
- Code is reused in 2+ places
- Logic is complex enough to deserve isolation
- Component serves a clear, single purpose
- Variations can be controlled via props

### Component Checklist
- [ ] TypeScript types for props
- [ ] JSDoc documentation with examples
- [ ] Storybook story with variants
- [ ] Unit tests (if logic is complex)
- [ ] Accessibility attributes (ARIA)
- [ ] Responsive design (mobile-first)
- [ ] Performance optimizations (memo if needed)

### Naming Conventions
- **PascalCase** for component files: `Button.tsx`
- **kebab-case** for utility files: `create-fade-up.ts`
- **camelCase** for functions: `getFeaturesData()`
- **SCREAMING_SNAKE_CASE** for constants: `ANIMATION_DURATION`
